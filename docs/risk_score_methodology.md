# 道路風險分數計算方法說明

## 概述

本系統對台灣每條道路計算一個 **0 到 1 的風險分數（Risk Score）**，作為自行車路線規劃的依據。
分數越高代表該路段歷史上的事故越嚴重，導航系統在規劃路線時會傾向繞開這些路段。

這份文件說明風險分數的計算原理、每個步驟的設計理由，以及涉及的技術細節。

---

## 資料來源

| 資料 | 說明 |
|------|------|
| 道路路段 | Geofabrik 提供的 OpenStreetMap 台灣路網（`gis_osm_roads_free_1.gpkg`） |
| 事故資料 | 台灣交通事故資料，已轉換為 GeoPackage 格式（`accidents_epsg3857.gpkg`） |

## 一、事故類型定義（法規依據）

台灣《道路交通事故處理辦法》將事故分為兩類：

| 類別 | 定義 |
|------|------|
| **A1 事故** | 造成人員**當場死亡**，或事故發生後 **24 小時以內死亡** |
| **A2 事故** | 造成人員**受傷**，或**超過 24 小時後死亡**（例如住院後不治） |

**重要說明**：A2 事故仍可能有死亡案例（只是死亡時間超過 24 小時），並非「A2 = 純受傷」。
A1 和 A2 的本質區別在於**急性致命程度**，而非是否有人死亡。

---

## 二、嚴重程度分數（Severity Score）

每筆事故依其類型計算嚴重程度：

```
A1 事故：severity = death_count × 3.0 + injury_count × 1.5
A2 事故：severity = death_count × 2.0 + injury_count × 1.0
```

### 各係數的理由

| 係數 | 值 | 理由 |
|------|-----|------|
| A1 死亡（`RISK_A1_DEATH_WEIGHT`） | 3.0 | 24 小時內死亡代表極高能量碰撞或嚴重撞擊，反映最嚴峻的道路危險 |
| A1 受傷（`RISK_A1_INJURY_WEIGHT`） | 1.5 | A1 事故中的受傷者處於更危險的事故情境，比 A2 受傷嚴重 |
| A2 死亡（`RISK_A2_DEATH_WEIGHT`） | 2.0 | 雖死亡時間較晚，仍屬嚴重事故，危險程度次於 A1 |
| A2 受傷（`RISK_A2_INJURY_WEIGHT`） | 1.0 | 基準值，所有受傷事故的最低嚴重程度 |

> **注意**：這些係數目前為暫定值，待取得 VSL（統計生命價值，Value of Statistical Life）相關研究資料後，將依照成本比例更新。VSL 研究通常以「防止一件死亡事故的社會成本」為基準，推算不同嚴重程度的相對比例。

---

## 三、時間衰減（Time Decay）

**問題**：10 年前的事故是否應該和去年的事故有同等的影響力？

道路環境會隨時間改善（拓寬、設置號誌、施工改善），舊事故的預測力下降。
因此，越久遠的事故，對當前危險度的貢獻越小。

### 計算方式（指數衰減）

```
decay_λ = ln(2) / half_life_years      # 半衰期換算成衰減率
years_ago = (今天 - 事故日期).days / 365.25
time_decay = e^(-decay_λ × years_ago)
```

**半衰期設定為 3 年**，意思是：

| 事故發生時間 | 時間權重 |
|------------|---------|
| 1 年前 | 79.4% |
| 3 年前 | 50.0%（半衰期） |
| 6 年前 | 25.0% |
| 10 年前 | 9.9% |
| 15 年前 | 3.1% |

### 為什麼是 3 年？

台灣道路工程的一般改善週期約 3–5 年（重大路口改善、自行車道建置等）。
3 年為保守估計，確保近期事故仍有足夠的影響力。未來可根據實際資料調整。

---

## 四、事故複合權重

每筆事故的最終複合權重：

```
weight_i = severity_i × time_decay_i
```

**範例**：

一件 2 年前的 A1 事故（1 人死亡，2 人受傷）：
```
severity = 1 × 3.0 + 2 × 1.5 = 6.0
time_decay = e^(-0.231 × 2) = 0.631
weight = 6.0 × 0.631 = 3.79
```

一件 5 年前的 A2 事故（0 人死亡，3 人受傷）：
```
severity = 0 × 2.0 + 3 × 1.0 = 3.0
time_decay = e^(-0.231 × 5) = 0.315
weight = 3.0 × 0.315 = 0.95
```

## 五、事故對應到道路（Spatial Snap）

每筆事故（點座標）需要找到它「屬於」哪一條道路（`osm_id`）。

### 為什麼需要這個步驟？

事故資料的 GPS 座標有定位誤差，通常不會精確落在道路線段上。
需要「找最近的道路」這個空間查詢步驟，才能把事故歸屬到正確的路段。

### 技術方法

使用 GeoPandas 的 `sjoin_nearest`（空間最近鄰居 Join）：

```python
joined = gpd.sjoin_nearest(
    accidents,   # 點幾何（事故位置）
    roads,       # 線幾何（道路路段）
    max_distance=20.0,   # 20 公尺以內才算命中
    distance_col="snap_distance_m",
)
```

**公差設定為 20 公尺**：
- 台灣事故資料的 GPS 定位誤差通常在 10–20 公尺以內
- 在 QGIS 中驗證：20m 公差下 99.51% 的事故可以對應到道路
- 超過 20m 的事故視為資料問題（可能記錄在停車場、建築物內），捨棄不計

### 兩個座標系統的轉換

事故資料和道路資料都轉換為 **EPSG:3857（公尺單位）**，確保「20 公尺」的距離計算正確。
（若使用 WGS84 經緯度，1 度在台灣緯度約等於 111 公里，無法直接用度數計算公尺距離。）

## 六、路段風險密度（Edge Risk Density）

將對應到同一路段的所有事故權重累加，除以路段長度（公里）：

```
edge_raw_risk[osm_id] = Σ weight_i  /  edge_length_km
```

### 為什麼要除以長度？

如果直接累加權重，一條 5 公里的道路會比一條 100 公尺的小巷累積更多分數，
但這不代表那條長路更危險——只是因為它比較長，經過的事故機會更多。

除以長度後得到「每公里的危險密度」，讓不同長度的路段可以公平比較。

---

## 七、正規化（Normalization）

將原始風險密度縮放到 [0, 1]，方便與路線長度組合計算：

```python
# 取非零路段的 P99 值作為截斷點
clip_value = percentile(非零路段風險密度, 99.0)

# 超過截斷點的路段統一設為 1.0
normalized_risk = min(raw_risk / clip_value, 1.0)
```

### 為什麼用 P99 而不是 max？

台灣某些特別危險的路口（如大型交叉路口、交流道附近）的事故數量遠高於其他路段。
如果用 `max` 作為除數：

- 假設 `max` 是某個異常危險路口的值 = 100
- 大多數路段的原始值可能在 2–10 之間
- 除以 100 後，這些路段的分數只有 0.02–0.10，幾乎都接近 0
- **結果**：所有路段看起來都安全，只有那個最危險路口是 1.0，失去區分度

用 P99（排除最高 1% 的極端值）：
- 99% 的路段在 [0, 1] 之間均勻分布
- 最高 1% 的極端危險路段統一視為 1.0
- **結果**：大多數路段有有意義的區分，導航可以有效區分「稍危險」vs「很危險」

---

## 八、與路線規劃的整合

路線規劃使用 **Dijkstra 最短路徑演算法**，邊的「距離」不只是實際距離，而是加入風險懲罰：

```
risk_weight = length_m × (1 + λ × normalized_risk)
```

其中 `λ`（lambda）是使用者可調整的風險迴避係數：

| λ 值 | 效果 |
|------|------|
| 0.0 | 完全不考慮風險，只找最短路線 |
| 0.5（預設） | 平衡距離與安全性 |
| 2.0 | 高度偏好安全路線，即使繞遠路也接受 |
| 5.0 | 最大程度迴避危險路段 |

**範例**：

一條 500 公尺的道路，risk_score = 0.8，λ = 1.0：
```
risk_weight = 500 × (1 + 1.0 × 0.8) = 500 × 1.8 = 900
```
等效於讓 Dijkstra 認為這條路有 900 公尺長，讓演算法傾向選擇實際上更長但更安全的替代路線。

## 九、完整流程圖

```
原始事故資料（點座標 + 日期 + 死亡/受傷人數 + 類型 A1/A2）
        │
        ▼
┌─────────────────────────────────────────┐
│  嚴重程度計算                            │
│  A1: deaths×3.0 + injuries×1.5         │
│  A2: deaths×2.0 + injuries×1.0         │
└─────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────┐
│  時間衰減                                │
│  half_life = 3年                        │
│  time_decay = e^(-0.231 × years_ago)   │
└─────────────────────────────────────────┘
        │
        ▼
   weight_i = severity × time_decay
        │
        ▼
┌─────────────────────────────────────────┐
│  空間 Snap（sjoin_nearest, 20m 公差）    │
│  每筆事故 → 對應 osm_id                  │
│  命中率：約 99%                          │
└─────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────┐
│  依 osm_id 累加 → 除以路段長度(km)       │
│  edge_risk = Σweight / length_km        │
└─────────────────────────────────────────┘
        │
        ▼
┌─────────────────────────────────────────┐
│  P99 截斷 → 正規化到 [0, 1]             │
│  normalized_risk = min(raw/P99, 1.0)   │
└─────────────────────────────────────────┘
        │
        ▼
   risk_scores = { "osm_id": 0.0~1.0, ... }
        │
        ▼
┌─────────────────────────────────────────┐
│  Dijkstra 邊權重                         │
│  risk_weight = length × (1 + λ × risk) │
└─────────────────────────────────────────┘
        │
        ▼
   安全優化路線
```

## 十、可調整參數摘要

所有參數集中在 `backend/app/config.py`（或 `.env` 檔案）：

| 參數 | 預設值 | 說明 |
|------|--------|------|
| `RISK_A1_DEATH_WEIGHT` | 3.0 | A1 死亡係數 |
| `RISK_A1_INJURY_WEIGHT` | 1.5 | A1 受傷係數 |
| `RISK_A2_DEATH_WEIGHT` | 2.0 | A2 死亡係數 |
| `RISK_A2_INJURY_WEIGHT` | 1.0 | A2 受傷基準係數 |
| `RISK_DECAY_HALF_LIFE_YEARS` | 3.0 | 時間衰減半衰期（年） |
| `RISK_CLIP_PERCENTILE` | 99.0 | 正規化截斷百分位 |
| `SNAP_TOLERANCE_M` | 20.0 | 事故 snap 最大距離（公尺） |
| `LAMBDA_DEFAULT` | 0.5 | 預設風險迴避係數 |

## 十一、已知限制與未來改善方向

1. **VSL 係數待研究**：A1/A2 的死亡/受傷係數目前為暫定值，應參考台灣或國際的 VSL 研究更新。

2. **半衰期設定**：3 年半衰期未經實證驗證，建議未來以模型驗證（比較不同半衰期設定下的預測準確度）。

3. **事故資料覆蓋率**：若事故資料不完整（某些年份缺漏），時間衰減可能產生偏差。

4. **道路改善未反映**：某條路段若近期已進行安全改善（如設置自行車道、加裝號誌），
   舊事故的風險仍然存在於分數中，無法即時反映改善效果。

5. **夜間/天氣因素**：目前未區分事故發生時間或天氣條件，未來可加入時段風險係數。
